Frequently asked questions about the CV builder

Is it really free?
Yes. Completely free. No subscriptions, no ads, no paid plans. The code is open source and your CV is hosted on GitHub Pages, a free service from GitHub.
Do I need to know how to code?
No. The editor is 100% visual. You fill in forms with your data and the CV is generated automatically. You don't need to touch a single line of code.
What is JSON Resume?
It's an open, standard format for resumes. It keeps your data separate from the design, so you can switch templates without losing anything.
Why do you use GitHub?
GitHub Pages is free, reliable and doesn't depend on us. Your data lives in your own account. If the app stopped existing, your CV would still be available.
Can I change the design later?
Yes. You can switch templates anytime. Your data stays intact. Only the design changes.
Can I download my CV as PDF?
Yes. From the browser you can print or save your CV as PDF with one click.
Where does my data live?
In your personal GitHub repository. Only you have write access. The app has no servers of its own to store your data.
What happens if I stop using the app?
Nothing. Your data is in your GitHub account. You can download the JSON, use another JSON Resume-compatible editor or keep updating manually.
Can I import a resume.json?
Yes. If you already have a resume.json file, you can import it directly and start editing.
Can I edit my CV after publishing?
Yes. Return to the editor, change what you need and republish. Your website updates instantly.
Can I delete my CV?
Yes. Delete your GitHub repository and your CV stops being available immediately.
Can I have multiple languages?
Yes. You can create your CV in Spanish, Italian or English. Switch languages instantly.
Do I need to install anything?
No. It works directly in the browser. You don't need to download or install any software.
What happens if GitHub changes?
You always have a copy of your data in standard format. You can migrate to any JSON Resume-compatible service.
Why is it better than a traditional editor?
Because it separates data from design. In Word or Canva, every change can break the format. Here, you change what you want and the design always stays intact.
